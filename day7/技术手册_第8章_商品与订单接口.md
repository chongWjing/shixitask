## 8.1 分页结果类 PageResult

"分页是电商系统最常见的功能。商品列表、订单列表都需要分页。我们先封装一个通用的分页结果类，后续所有分页接口都复用它。"

在 `com.ecommerce.entity` 包下创建 `PageResult.java`：

```java
package com.ecommerce.entity;

import lombok.Data;

import java.util.List;

/**
 * 分页结果类
 */
@Data
public class PageResult<T> {

    /**
     * 当前页数据列表
     */
    private List<T> list;

    /**
     * 总记录数
     */
    private long total;

    /**
     * 当前页码
     */
    private int page;

    /**
     * 每页条数
     */
    private int size;

    public PageResult() {
    }

    public PageResult(int page, int size, long total, List<T> list) {
        this.page = page;
        this.size = size;
        this.total = total;
        this.list = list;
    }
}
```

**预期效果：** PageResult类使用Lombok的@Data注解自动生成getter/setter，封装分页查询的结果，前端可以根据total判断是否还有下一页。

### 8.2.3 创建ProductService

在 `com.ecommerce.service` 包下创建 `ProductService.java`：

```java
package com.ecommerce.service;

import com.ecommerce.entity.PageResult;
import com.ecommerce.entity.Product;
import com.ecommerce.mapper.ProductMapper;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;

import java.util.Date;
import java.util.List;

/**
 * 商品服务类
 */
@Service
public class ProductService {

    @Autowired
    private ProductMapper productMapper;

    /**
     * 分页查询商品列表
     *
     * @param page 当前页码（从1开始）
     * @param size 每页条数
     * @return 分页结果
     */
    public PageResult<Product> getProductList(int page, int size) {
        // 第一步：参数校验和默认值处理
        if (page < 1) {
            page = 1;
        }
        if (size < 1) {
            size = 10;
        }

        // 第二步：计算偏移量（MySQL的LIMIT从0开始）
        int offset = (page - 1) * size;

        // 第三步：查询当前页数据
        List<Product> list = productMapper.findByPage(offset, size);

        // 第四步：查询总数
        long total = productMapper.count();

        // 第五步：封装分页结果
        return new PageResult<>(page, size, total, list);
    }

    /**
     * 查询商品详情
     *
     * @param id 商品ID
     * @return 商品信息
     */
    public Product getProductById(Integer id) {
        if (id == null) {
            return null;
        }
        return productMapper.findById(id);
    }

    /**
     * 搜索商品（LIKE模糊查询，第13章升级为FULLTEXT全文索引）
     *
     * @param keyword 搜索关键词
     * @return 匹配的商品列表
     */
    public List<Product> searchProduct(String keyword) {
        if (keyword == null || keyword.trim().isEmpty()) {
            return productMapper.findByPage(0, 20);
        }
        return productMapper.searchByKeyword(keyword.trim());
    }

    /**
     * 按分类分页查询商品（第11章补充）
     */
    public PageResult<Product> getProductListByCategory(int page, int size, String category) {
        if (page < 1) {
            page = 1;
        }
        if (size < 1) {
            size = 10;
        }
        int offset = (page - 1) * size;
        List<Product> list = productMapper.findByCategoryPage(offset, size, category);
        long total = productMapper.countByCategory(category);
        return new PageResult<>(page, size, total, list);
    }

    /**
     * 全文索引搜索商品（第13章补充）
     */
    public List<Product> searchProductFulltext(String keyword) {
        if (keyword == null || keyword.trim().isEmpty()) {
            return productMapper.findByPage(0, 20);
        }
        return productMapper.searchByKeywordFulltext(keyword.trim());
    }

    /**
     * 新增商品（第9章补充）
     *
     * @param product 商品信息
     * @return 操作结果消息
     */
    public String addProduct(Product product) {
        if (product.getStatus() == null) {
            product.setStatus(1);
        }
        if (product.getStock() == null) {
            product.setStock(0);
        }
        if (product.getSales() == null) {
            product.setSales(0);
        }
        product.setCreateTime(new Date());

        int rows = productMapper.insert(product);
        if (rows > 0) {
            return "新增商品成功";
        }
        return "新增商品失败";
    }

    /**
     * 修改商品（第9章补充）
     *
     * @param product 商品信息（必须包含id）
     * @return 操作结果消息
     */
    public String updateProduct(Product product) {
        Product existProduct = productMapper.findById(product.getId());
        if (existProduct == null) {
            return "商品不存在";
        }
        int rows = productMapper.update(product);
        if (rows > 0) {
            return "修改商品成功";
        }
        return "修改商品失败";
    }

    /**
     * 修改商品状态/上下架（第9章补充）
     *
     * @param id     商品ID
     * @param status 状态：0-下架，1-上架
     * @return 操作结果消息
     */
    public String updateProductStatus(Integer id, Integer status) {
        if (status == null || (status != 0 && status != 1)) {
            return "状态值无效，0-下架，1-上架";
        }
        Product existProduct = productMapper.findById(id);
        if (existProduct == null) {
            return "商品不存在";
        }
        int rows = productMapper.updateStatus(id, status);
        if (rows > 0) {
            return status == 1 ? "商品已上架" : "商品已下架";
        }
        return "修改状态失败";
    }
}
```

**预期效果：** getProductList方法支持分页查询（参数为int类型），page从1开始，自动计算offset。getProductById方法根据ID查询单个商品详情。searchProduct方法根据关键词搜索商品名称和描述。addProduct/updateProduct/updateProductStatus方法在第9章管理接口中使用，返回String类型的结果消息。

## 8.3 ProductController编写

"商品控制器提供三个GET接口，用于查询商品列表、商品详情和搜索商品。这三个接口不需要登录也可以访问——毕竟用户逛商城的时候，还没登录也想看到商品。"

在 `com.ecommerce.controller` 包下创建 `ProductController.java`：

```java
package com.ecommerce.controller;

import com.ecommerce.entity.PageResult;
import com.ecommerce.entity.Product;
import com.ecommerce.entity.Result;
import com.ecommerce.service.ProductService;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.web.bind.annotation.*;

import java.util.List;

/**
 * 商品控制器
 */
@RestController
@RequestMapping("/api/product")
public class ProductController {

    @Autowired
    private ProductService productService;

    /**
     * 查询商品列表（分页）
     * GET /api/product/list?page=1&size=10
     *
     * @param page 当前页码
     * @param size 每页条数
     * @return 分页商品列表
     */
    @GetMapping("/list")
    public Result<PageResult<Product>> getProductList(
            @RequestParam(defaultValue = "1") int page,
            @RequestParam(defaultValue = "10") int size) {
        PageResult<Product> pageResult = productService.getProductList(page, size);
        return Result.success("查询成功", pageResult);
    }

    /**
     * 查询商品详情
     * GET /api/product/{id}
     *
     * @param id 商品ID
     * @return 商品详情
     */
    @GetMapping("/{id}")
    public Result<Product> getProductById(@PathVariable Integer id) {
        Product product = productService.getProductById(id);
        if (product == null) {
            return Result.error(400, "商品不存在");
        }
        return Result.success("查询成功", product);
    }

    /**
     * 搜索商品
     * GET /api/product/search?keyword=xxx
     *
     * @param keyword 搜索关键词
     * @return 匹配的商品列表
     */
    @GetMapping("/search")
    public Result<List<Product>> searchProduct(@RequestParam String keyword) {
        List<Product> list = productService.searchProduct(keyword);
        return Result.success("搜索成功", list);
    }
}
```

"注意！`/api/product/list`、`/api/product/{id}` 和 `/api/product/search` 这三个接口是公开的，不需要登录就能访问。但是拦截器会把它们拦截住。我们需要回到WebMvcConfig中，把商品查询接口排除掉。"

### 8.3.1 更新WebMvcConfig排除商品接口

修改 `WebMvcConfig.java`，在 `excludePathPatterns` 中添加商品查询路径：

```java
package com.ecommerce.config;

import com.ecommerce.interceptor.LoginInterceptor;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.context.annotation.Configuration;
import org.springframework.web.servlet.config.annotation.InterceptorRegistry;
import org.springframework.web.servlet.config.annotation.WebMvcConfigurer;

/**
 * Web MVC配置类：注册拦截器
 */
@Configuration
public class WebMvcConfig implements WebMvcConfigurer {

    @Autowired
    private LoginInterceptor loginInterceptor;

    /**
     * 注册拦截器
     */
    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        registry.addInterceptor(loginInterceptor)
                .addPathPatterns("/api/**")
                .excludePathPatterns(
                        "/api/user/login",
                        "/api/user/register",
                        "/api/test/**",
                        "/api/product/list",          // 商品列表
                        "/api/product/search",        // 商品搜索
                        "/api/product/*"              // 商品详情（/api/product/{id}）
                );
    }
}
```

**预期效果：** 商品列表、商品详情和商品搜索接口无需登录即可访问，返回商品的完整信息。

### 8.4.3 创建OrderService

在 `com.ecommerce.service` 包下创建 `OrderService.java`：

```java
package com.ecommerce.service;

import com.ecommerce.entity.Order;
import com.ecommerce.entity.PageResult;
import com.ecommerce.entity.Product;
import com.ecommerce.mapper.OrderMapper;
import com.ecommerce.mapper.ProductMapper;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

import java.math.BigDecimal;
import java.util.Date;
import java.util.List;

/**
 * 订单服务类
 */
@Service
public class OrderService {

    @Autowired
    private OrderMapper orderMapper;

    @Autowired
    private ProductMapper productMapper;

    /**
     * 创建订单
     * 使用@Transactional保证事务一致性：库存扣减和订单插入要么同时成功，要么同时回滚
     *
     * @param userId          用户ID
     * @param productId       商品ID
     * @param quantity        购买数量
     * @param receiverName    收货人姓名
     * @param receiverPhone   收货人手机号
     * @param receiverAddress 收货地址
     * @return 创建的订单对象
     */
    @Transactional
    public Order createOrder(Integer userId, Integer productId, Integer quantity,
                             String receiverName, String receiverPhone,
                             String receiverAddress) {
        // 第一步：验证商品是否存在
        Product product = productMapper.findById(productId);
        if (product == null) {
            throw new RuntimeException("商品不存在");
        }

        // 第二步：验证商品是否上架
        if (product.getStatus() == null || product.getStatus() != 1) {
            throw new RuntimeException("商品已下架");
        }

        // 第三步：验证库存是否充足
        if (product.getStock() < quantity) {
            throw new RuntimeException("库存不足，当前库存：" + product.getStock());
        }

        // 第四步：扣减库存（使用条件更新：WHERE stock >= quantity，防止超卖）
        int stockRows = productMapper.decreaseStock(productId, quantity);
        if (stockRows == 0) {
            throw new RuntimeException("库存不足，下单失败");
        }

        // 第五步：增加销量
        productMapper.increaseSales(productId, quantity);

        // 第六步：生成订单号（ORD + 时间戳）
        String orderNo = "ORD" + System.currentTimeMillis();

        // 第七步：冗余商品信息到订单（防止商品信息修改后影响历史订单）
        Order order = new Order();
        order.setOrderNo(orderNo);
        order.setUserId(userId);
        order.setProductId(productId);
        order.setProductName(product.getName());
        order.setProductImage(product.getImage());
        order.setProductPrice(product.getPrice());
        order.setQuantity(quantity);
        // 计算总价 = 商品单价 × 数量
        order.setTotalPrice(product.getPrice().multiply(new BigDecimal(quantity)));
        order.setReceiverName(receiverName);
        order.setReceiverPhone(receiverPhone);
        order.setReceiverAddress(receiverAddress);
        order.setStatus(0);    // 待付款
        order.setCreateTime(new Date());
        order.setUpdateTime(new Date());

        // 第八步：插入订单记录
        int orderRows = orderMapper.insert(order);
        if (orderRows == 0) {
            throw new RuntimeException("创建订单失败");
        }

        return order;
    }

    /**
     * 查询我的订单
     *
     * @param userId 用户ID
     * @return 订单列表
     */
    public List<Order> getMyOrders(Integer userId) {
        return orderMapper.findByUserId(userId);
    }

    /**
     * 查询所有订单（分页，第9章管理后台使用）
     *
     * @param page 当前页码
     * @param size 每页条数
     * @return 分页结果
     */
    public PageResult<Order> getAllOrders(int page, int size) {
        if (page < 1) {
            page = 1;
        }
        if (size < 1) {
            size = 10;
        }
        int offset = (page - 1) * size;
        List<Order> list = orderMapper.findByPage(offset, size);
        long total = orderMapper.count();
        return new PageResult<>(page, size, total, list);
    }

    /**
     * 修改订单状态（第9章管理后台使用）
     *
     * @param id     订单ID
     * @param status 订单状态：0待付款 1已付款 2已发货 3已完成 4已取消
     */
    public void updateOrderStatus(Integer id, Integer status) {
        if (status == null || status < 0 || status > 4) {
            throw new RuntimeException("订单状态无效，0-待付款 1-已付款 2-已发货 3-已完成 4-已取消");
        }
        orderMapper.updateStatus(id, status);
    }

    /**
     * 按状态分页查询订单（第9章管理后台使用）
     *
     * @param page   当前页码
     * @param size   每页条数
     * @param status 订单状态
     * @return 分页结果
     */
    public PageResult<Order> getAllOrdersByStatus(int page, int size, Integer status) {
        if (page < 1) {
            page = 1;
        }
        if (size < 1) {
            size = 10;
        }
        int offset = (page - 1) * size;
        List<Order> list = orderMapper.findByPageWithStatus(offset, size, status);
        long total = orderMapper.countByStatus(status);
        return new PageResult<>(page, size, total, list);
    }
}
```

**预期效果：** createOrder方法在事务中完成下单流程，包括商品验证、库存扣减、销量增加、订单号生成、商品信息冗余、总价计算和订单插入。库存不足或商品不存在时抛出RuntimeException，由Spring事务管理器回滚。getMyOrders方法直接返回订单列表。getAllOrders和updateOrderStatus方法在第9章管理接口中使用。

## 8.5 OrderController编写

"订单接口需要登录后才能使用。用户必须先登录获取Token，然后在请求头中携带Token才能创建订单和查看订单。"

在 `com.ecommerce.controller` 包下创建 `OrderController.java`：

```java
package com.ecommerce.controller;

import com.ecommerce.entity.Order;
import com.ecommerce.entity.Result;
import com.ecommerce.service.OrderService;
import jakarta.servlet.http.HttpServletRequest;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.web.bind.annotation.*;

import java.util.List;
import java.util.Map;

/**
 * 订单控制器
 */
@RestController
@RequestMapping("/api/order")
public class OrderController {

    @Autowired
    private OrderService orderService;

    /**
     * 创建订单
     * POST /api/order/create
     * 需要登录
     *
     * @param params  包含productId、quantity、receiverName、receiverPhone、receiverAddress
     * @param request HTTP请求对象
     * @return 创建结果
     */
    @PostMapping("/create")
    public Result<Order> createOrder(@RequestBody Map<String, Object> params,
                                      HttpServletRequest request) {
        // 从request属性获取userId（由拦截器设置）
        Integer userId = (Integer) request.getAttribute("userId");

        // 获取参数
        Integer productId = (Integer) params.get("productId");
        Integer quantity = (Integer) params.get("quantity");
        String receiverName = (String) params.get("receiverName");
        String receiverPhone = (String) params.get("receiverPhone");
        String receiverAddress = (String) params.get("receiverAddress");

        // 参数校验
        if (productId == null) {
            return Result.error(400, "商品ID不能为空");
        }
        if (quantity == null || quantity < 1) {
            return Result.error(400, "购买数量不能为空且必须大于0");
        }
        if (receiverName == null || receiverName.trim().isEmpty()) {
            return Result.error(400, "收货人姓名不能为空");
        }
        if (receiverPhone == null || receiverPhone.trim().isEmpty()) {
            return Result.error(400, "收货人手机号不能为空");
        }
        if (receiverAddress == null || receiverAddress.trim().isEmpty()) {
            return Result.error(400, "收货地址不能为空");
        }

        // 调用Service创建订单
        try {
            Order order = orderService.createOrder(userId, productId, quantity,
                    receiverName, receiverPhone, receiverAddress);
            return Result.success("下单成功", order);
        } catch (RuntimeException e) {
            return Result.error(400, e.getMessage());
        }
    }

    /**
     * 查询我的订单
     * GET /api/order/my
     * 需要登录
     *
     * @param request HTTP请求对象
     * @return 订单列表
     */
    @GetMapping("/my")
    public Result<List<Order>> getMyOrders(HttpServletRequest request) {
        // 从request属性获取userId（由拦截器设置）
        Integer userId = (Integer) request.getAttribute("userId");

        List<Order> orders = orderService.getMyOrders(userId);
        return Result.success("查询成功", orders);
    }
}
```

**预期效果：** 创建订单接口需要登录，从Token中获取userId，校验参数后调用Service。Service返回Order对象或抛出RuntimeException，Controller通过try-catch捕获异常并返回错误信息。查询我的订单接口也需登录，返回当前用户的所有订单。所有代码使用`jakarta.servlet`（SpringBoot 3.x规范）。

## 常见问题

| 序号 | 问题描述 | 可能原因 | 解决方案 |
|------|----------|----------|----------|
| 1 | 商品列表返回空数组 | 数据库中没有上架商品 | 插入测试数据，确保status=1 |
| 2 | 创建订单时提示"商品不存在" | productId错误或商品未上架 | 检查商品ID是否正确，检查商品status是否为1 |
| 3 | 创建订单时提示"库存不足" | 商品库存小于购买数量 | 检查数据库中商品stock字段值 |
| 4 | 创建订单后库存没有减少 | decreaseStock的WHERE条件不满足 | 确认SQL中 `stock >= #{quantity}` 条件 |
| 5 | 订单中商品名称为null | 下单时未冗余商品信息 | 检查createOrder中是否设置了productName等字段 |
| 6 | 总价计算精度丢失 | 使用了double而不是BigDecimal | 必须使用BigDecimal进行金额计算 |
| 7 | 创建订单部分成功部分失败 | 缺少事务注解 | 确保createOrder方法上有@Transactional注解 |
| 8 | 商品接口被拦截器拦截返回401 | 拦截器未排除商品路径 | 检查WebMvcConfig中是否排除了 /api/product/list、/api/product/search 和 /api/product/* |
| 9 | 商品搜索返回空结果 | LIKE查询区分大小写或关键词不匹配 | 检查数据库中商品名称和描述是否包含关键词，第13章升级FULLTEXT |
| 10 | jakarta.servlet编译报错 | 使用了javax.servlet | SpringBoot 3.x必须使用jakarta.servlet |

## 本章小结

本章完成了商品模块和订单模块的核心功能：

1. **PageResult分页类**：使用@Data注解，包含list、total、page、size四个字段
2. **Product实体**：使用@Data注解，包含originalPrice字段
3. **ProductMapper**：实现分页查询、详情查询、库存扣减、销量增加和LIKE关键词搜索
4. **ProductService**：实现了商品列表分页查询、商品详情查询和关键词搜索
5. **ProductController**：提供三个GET接口——商品列表、商品详情和商品搜索，无需登录即可访问
6. **Order实体**：使用@Data注解，收货信息使用receiverName/receiverPhone/receiverAddress
7. **OrderService**：实现了创建订单（8步流程：验证商品→检查上架→检查库存→扣减库存→增加销量→生成订单号→冗余商品信息→插入订单，返回Order对象，异常时抛出RuntimeException）和查询我的订单。还包含第9章管理后台使用的getAllOrders、updateOrderStatus和getAllOrdersByStatus方法
8. **OrderController**：提供创建订单和查询我的订单两个接口，需要登录，使用jakarta.servlet。createOrder通过try-catch捕获Service层RuntimeException

**核心设计要点：**
- **防超卖**：decreaseStock使用 `WHERE stock >= #{quantity}` 条件更新
- **事务一致性**：createOrder方法使用 `@Transactional` 保证库存扣减和订单插入的原子性
- **信息冗余**：订单中保存商品名称、图片、价格，防止商品信息修改后影响历史订单
- **金额计算**：使用BigDecimal避免浮点精度问题
- **Jakarta EE**：SpringBoot 3.x使用jakarta.servlet替代javax.servlet
- **表名规范**：使用product/orders而非t_product/t_order
- **LIKE搜索**：当前使用简单LIKE模糊查询，第13章将升级为FULLTEXT全文索引

**下一章预告：** 第9章将实现管理后台接口，包括商品管理（增删改查、上下架）和订单管理（查询所有订单、修改订单状态），并加入管理员权限验证。
