## 4.3 页面跳转与数据传递（15分钟）

"我们来梳理一下页面跳转的流程——"

**（板书）**

```
页面跳转流程：

首页(/)
  │ 点击商品卡片
  ↓
商品详情页(/product/:id)
  │ 点击"🛒 立即购买"
  ├─ 未登录 → 登录页(/login?redirect=/order/1)
  │              │ 登录成功
  │              ↓
  │           下单页(/order/1)
  └─ 已登录 → 下单页(/order/1)
```

"路由传参用了两种方式——"

| 方式 | 代码 | 使用场景 |
|------|------|---------|
| 动态路由参数 | `/product/:id` → `this.$route.params.id` | 商品详情页、下单页，ID是URL的一部分 |
| Query参数 | `/login?redirect=/order/1` → `this.$route.query.redirect` | 登录页，传递登录后要跳转的目标路径 |

"路由守卫处理了登录检查——在router/index.js中，我们给下单页设置了 `meta: { requiresAuth: true }`，未登录会自动跳转到登录页。"

"下单页路由从 `/order?productId=1` 改为 `/order/:id` 的原因——"

| 旧写法 | 新写法 | 为什么改 |
|--------|--------|----------|
| `/order?productId=1` | `/order/1` | 统一规范规定 `/order/:id`，RESTful风格更清晰 |
| `this.$route.query.productId` | `this.$route.params.id` | params从路径段取值，query从问号后取值 |

## 本章常见问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 商品列表不是4列 | grid-template-columns设置错误 | 确保设置为repeat(4, 1fr)，容器宽度1200px |
| 商品名称超出没有省略号 | 缺少-webkit-line-clamp样式 | 添加display:-webkit-box和-webkit-line-clamp:2 |
| 原价没有划线效果 | 缺少text-decoration | 给.original-price添加text-decoration: line-through |
| 详情页商品数据不显示 | 路由参数获取错误 | 检查this.$route.params.id是否能正确获取 |
| 分类筛选后列表不变 | 没有重置分页 | 在onCategoryClick中重置currentPage为1 |
| 详情页左右分栏不生效 | flex布局设置错误 | 确保detail-main设置display:flex，左侧固定450px宽度 |
| 价格区域没有渐变背景 | 背景样式未设置 | 确保price-section设置linear-gradient(#FFF5F5, #FFE0E0) |
| 面包屑导航不显示 | 未引入el-breadcrumb | 确保使用el-breadcrumb和el-breadcrumb-item组件 |
