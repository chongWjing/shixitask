# 第11章：后台管理员登录与主页

## 本章目标

> "同学们，前面我们做的都是商城端的页面。从这章开始，我们要进入后台管理系统的开发！后台管理系统是给管理员用的，用来管理商品、查看订单、统计数据。它跑在PC浏览器上，我们用ElementPlus组件库来构建，和商城端同属一个Vue3项目，统一使用ElementPlus PC端组件。"

完成本章后，你将能够：
- 使用Vue3+ElementPlus构建PC端后台管理页面
- 实现管理员登录验证（判断role===1）
- 构建后台主页框架（侧边栏+导航栏+内容区+路由切换）
- 开发数据概览统计接口，并增加管理员权限验证

## 11.1 后台管理系统说明

> "后台管理系统和商城端是同一个Vue3项目，统一使用ElementPlus组件库。通过路由区分：商城页面走 `/shop/*` 路径使用ShopLayout包裹，后台管理走 `/admin/*` 路径使用AdminLayout包裹。"

**技术选型：**
- 页面框架：Vue3 + Element Plus 2.7+（桌面端UI组件库，全项目统一）
- 路由：Vue Router 4.x（/admin/* 路径前缀，使用AdminLayout包裹）
- 状态管理：localStorage（token存储）
- 页面位置：`frontend/src/views/admin/` 目录
- 访问方式：`http://localhost:5173/admin/login`

**页面结构：**
```
frontend/src/views/
├── ShopLayout.vue           # 商城布局页
├── Home.vue                 # 商城首页
├── Login.vue                # 商城登录页
├── ProductDetail.vue        # 商品详情页
├── Order.vue                # 下单页
├── MyOrders.vue             # 我的订单页
└── admin/
    ├── Login.vue            # 管理员登录页
    ├── Layout.vue           # 后台布局（侧边栏+导航栏+内容区）
    ├── Dashboard.vue        # 首页概览（统计卡片）
    ├── ProductManage.vue    # 商品管理
    ├── OrderManage.vue      # 订单管理
    └── AiQuery.vue          # AI智能查询
```

### 步骤2：创建登录页面组件

创建文件 `frontend/src/views/admin/Login.vue`：

```vue
<template>
  <!-- 管理员登录页面 -->
  <div class="login-container">
    <!-- 左侧装饰区 -->
    <div class="login-left">
      <el-icon :size="80" color="#fff"><Shop /></el-icon>
      <h1>花之恋鲜花电商</h1>
      <p>后台管理系统<br>高效管理商品与订单<br>数据驱动运营决策</p>
    </div>

    <!-- 右侧登录表单 -->
    <div class="login-right">
      <el-card class="login-card" shadow="never">
        <h2>管理员登录</h2>
        <p class="subtitle">请输入管理员账号和密码</p>

        <!-- 登录表单 -->
        <el-form ref="loginFormRef" :model="loginForm" :rules="loginRules" @submit.prevent="handleLogin">
          <el-form-item prop="username">
            <el-input
              v-model="loginForm.username"
              placeholder="用户名"
              prefix-icon="User"
              size="large"
            />
          </el-form-item>
          <el-form-item prop="password">
            <el-input
              v-model="loginForm.password"
              type="password"
              placeholder="密码"
              prefix-icon="Lock"
              size="large"
              show-password
              @keyup.enter="handleLogin"
            />
          </el-form-item>
          <el-form-item>
            <el-button
              type="primary"
              size="large"
              :loading="loading"
              style="width: 100%; background-color: #C71526; border-color: #C71526;"
              @click="handleLogin"
            >
              {{ loading ? '登录中...' : '登 录' }}
            </el-button>
          </el-form-item>
        </el-form>
      </el-card>
    </div>
  </div>
</template>

<script>
import { login } from '../../api/user'

export default {
  name: 'AdminLogin',
  data() {
    return {
      // 登录表单数据
      loginForm: {
        username: '',
        password: ''
      },
      // 表单验证规则
      loginRules: {
        username: [
          { required: true, message: '请输入用户名', trigger: 'blur' }
        ],
        password: [
          { required: true, message: '请输入密码', trigger: 'blur' }
        ]
      },
      // 登录加载状态
      loading: false
    }
  },
  methods: {
    // 处理登录
    async handleLogin() {
      // 表单验证
      try {
        await this.$refs.loginFormRef.validate()
      } catch {
        return
      }

      this.loading = true
      try {
        // 调用登录接口：POST /api/user/login
        const result = await login(this.loginForm)
        if (result.code === 200) {
          // 判断是否为管理员（role === 1）
          if (result.data.role === 1) {
            // 将token存入localStorage
            localStorage.setItem('token', result.data.token)
            localStorage.setItem('username', result.data.username)
            localStorage.setItem('role', result.data.role)
            // 跳转到后台主页
            this.$router.push('/admin/dashboard')
          } else {
            // 非管理员，不允许进入后台
            this.$message.error('您不是管理员，无法登录后台！')
          }
        } else {
          // 登录失败
          this.$message.error(result.message || '登录失败，请检查用户名和密码')
        }
      } catch (error) {
        console.error('登录请求失败：', error)
        this.$message.error('网络错误，请稍后重试')
      } finally {
        this.loading = false
      }
    }
  },
  created() {
    // 页面加载时检查是否已登录
    const token = localStorage.getItem('token')
    const role = localStorage.getItem('role')
    if (token && role === '1') {
      // 已登录且是管理员，直接跳转主页
      this.$router.push('/admin/dashboard')
    }
  }
}
</script>
```

> "注意几个关键点：第一，使用 `el-card`、`el-form`、`el-input`、`el-button` 这些 Element Plus 组件；第二，调用 `import { login } from '../../api/user'` 统一接口调用；第三，token 存到 `localStorage`，商城端和管理端统一；第四，登录成功后用 `this.$router.push('/admin/dashboard')` 做路由跳转；第五，左侧装饰区使用花之恋品牌主题色 #C71526 到 #8B0A1A 的渐变，登录按钮也是 #C71526 主题色。"

**预期效果：**
- 访问 `http://localhost:5173/admin/login` 显示登录页面

```vue
<script>
import { getStatsOverview } from '../../api/admin'

export default {
  name: 'AdminDashboard',
  data() {
    return {
      // 统计数据
      stats: {
        productCount: '--',
        todayOrderCount: '--',
        totalSales: '--',
        userCount: '--'
      }
    }
  },
  computed: {
    adminName() {
      return localStorage.getItem('username') || '管理员'
    },
    todayStr() {
      const now = new Date()
      const weekDays = ['星期日', '星期一', '星期二', '星期三', '星期四', '星期五', '星期六']
      return now.getFullYear() + '年' + (now.getMonth() + 1) + '月' + now.getDate() + '日 ' + weekDays[now.getDay()]
    }
  },
  methods: {
    // 加载统计数据
    async loadStats() {
      try {
        // 调用后端接口：GET /api/admin/stats/overview（需管理员权限）
        const result = await getStatsOverview()
        if (result.code === 200) {
          const data = result.data
          this.stats = {
            productCount: data.productCount || 0,
            todayOrderCount: data.todayOrderCount || 0,
            totalSales: data.totalSales || '0.00',
            userCount: data.userCount || 0
          }
        } else if (result.code === 401) {
          // token失效，跳转登录
          localStorage.removeItem('token')
          localStorage.removeItem('username')
          localStorage.removeItem('role')
          this.$router.push('/admin/login')
        }
      } catch (error) {
        console.error('加载统计数据失败：', error)
      }
    }
  },
  created() {
    this.loadStats()
  }
}
</script>
```

## 11.4 AdminStatsController（含管理员权限验证）

> "后台主页需要展示统计数据，我们需要新增一个Controller来提供统计接口。这个接口只给管理员用，必须验证token和管理员角色！没有权限验证的接口是不安全的——任何人都能直接调用接口获取数据。"

### 步骤5：创建AdminStatsController

在 `com.ecommerce.controller` 包下创建 `AdminStatsController.java`：

```java
package com.ecommerce.controller;

import com.ecommerce.common.Result;
import com.ecommerce.entity.Order;
import com.ecommerce.entity.Product;
import com.ecommerce.entity.User;
import com.ecommerce.mapper.OrderMapper;
import com.ecommerce.mapper.ProductMapper;
import com.ecommerce.mapper.UserMapper;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;

import jakarta.servlet.http.HttpServletRequest;
import java.math.BigDecimal;
import java.time.LocalDate;
import java.util.HashMap;
import java.util.List;
import java.util.Map;

/**
 * 后台统计控制器
 * 提供数据概览、趋势分析等统计接口
 * 所有接口需要管理员权限（role === 1）
 */
@RestController
@RequestMapping("/api/admin/stats")
public class AdminStatsController {

    @Autowired
    private ProductMapper productMapper;

    @Autowired
    private OrderMapper orderMapper;

    @Autowired
    private UserMapper userMapper;

    /**
     * 数据概览
     * 需要管理员权限验证：从请求头获取token，解析userId，查询用户role==1才允许访问
     * 返回：商品总数、今日订单数、总销售额、用户总数
     */
    @GetMapping("/overview")
    public Result overview(HttpServletRequest request) {
        // ========== 管理员权限验证 ==========
        // 从请求头获取token
        String token = request.getHeader("Authorization");
        if (token == null || token.isEmpty()) {
            return Result.error(401, "未登录，请先登录");
        }

        // 从token中解析userId（token格式为 "userId_时间戳"）
        Long userId = null;
        try {
            String[] parts = token.split("_");
            userId = Long.parseLong(parts[0]);
        } catch (Exception e) {
            return Result.error(401, "token无效，请重新登录");
        }

        // 查询用户，验证是否为管理员（role == 1）
        User user = userMapper.selectById(userId);
        if (user == null) {
            return Result.error(401, "用户不存在，请重新登录");
        }
        if (user.getRole() == null || user.getRole() != 1) {
            return Result.error(403, "无管理员权限，禁止访问");
        }

        // ========== 统计数据 ==========
        Map<String, Object> data = new HashMap<>();

        // 商品总数
        List<Product> allProducts = productMapper.selectAll();
        data.put("productCount", allProducts.size());

        // 今日订单数
        List<Order> allOrders = orderMapper.selectAll();
        String today = LocalDate.now().toString();
        long todayOrderCount = allOrders.stream()
                .filter(order -> order.getCreateTime() != null
                        && order.getCreateTime().toString().startsWith(today))
                .count();
        data.put("todayOrderCount", todayOrderCount);

        // 总销售额（统计状态>=1的订单）
        BigDecimal totalSales = allOrders.stream()
                .filter(order -> order.getStatus() != null && order.getStatus() >= 1)
                .map(order -> {
                    if (order.getTotalPrice() != null) {
                        return order.getTotalPrice();
                    }
                    return BigDecimal.ZERO;
                })
                .reduce(BigDecimal.ZERO, BigDecimal::add);
        data.put("totalSales", totalSales.toString());

        // 用户总数
        List<User> allUsers = userMapper.selectAll();
        data.put("userCount", allUsers.size());

        return Result.success(data);
    }
}
```

## 11.6 页面权限保护

> "后台页面必须做权限保护！不能让用户直接输入URL就进入后台。我们通过Vue Router的路由守卫来实现，而不是在每个页面中单独检查。"

在 `frontend/src/router/index.js` 中配置路由守卫：

```javascript
// 路由守卫：管理后台需要管理员登录
router.beforeEach((to, from, next) => {
  const token = localStorage.getItem('token')
  const role = localStorage.getItem('role')

  // 管理后台需要管理员登录
  if (to.meta.requiresAdmin && (!token || role !== '1')) {
    next('/admin/login')
    return
  }

  next()
})
```

路由配置中需要添加 `meta` 标记：

```javascript
{
  path: '/admin',
  component: AdminLayout,
  meta: { requiresAdmin: true },
  children: [
    { path: 'dashboard', component: Dashboard, meta: { requiresAdmin: true } },
    { path: 'product', component: ProductManage, meta: { requiresAdmin: true } },
    { path: 'order', component: OrderManage, meta: { requiresAdmin: true } },
    { path: 'ai', component: AiQuery, meta: { requiresAdmin: true } }
  ]
}
```

> "这种方案比之前每个页面中单独检查 `localStorage.getItem('token')` 更优雅——只需要在路由守卫中统一处理，所有带 `requiresAdmin` 标记的路由都会自动拦截未登录和非管理员用户。"

## 常见问题

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| 访问/admin/login显示空白 | 路由未配置或组件未注册 | 检查router/index.js中是否有/admin/login路由 |
| 登录提示"您不是管理员" | 用户的role不是1 | 数据库中修改用户role为1：`UPDATE user SET role=1 WHERE username='admin'` |
| 登录后role读取为undefined | 后端返回的是user对象嵌套 | 后端实际返回 `result.data.user.role`，需改为对应取值方式 |
| 登录后跳转dashboard又跳回登录 | localStorage写入失败或路由守卫判断有误 | 检查localStorage中token和role是否正确写入，role是否为字符串'1' |
| 统计卡片显示"--" | 后端接口未返回数据或接口报错 | 检查Network面板，确认/api/admin/stats/overview返回正常 |
| 统计卡片显示0 | 数据库中没有数据 | 往各表中插入测试数据（鲜花商品等） |
| ElementPlus样式不生效 | 未引入ElementPlus | 检查main.js中是否正确引入ElementPlus |
| 左侧菜单点击没反应 | 路由未配置 | 检查router中是否有对应路由路径 |
| 直接输入/admin/dashboard能进入 | 路由守卫未生效 | 确认路由守卫中meta.requiresAdmin判断是否正确 |
| AdminStatsController启动报错 | Mapper缺少selectAll方法 | 补充Mapper接口和XML中的selectAll |
| 接口返回401 | token未携带或无效 | 检查请求头Authorization是否有值，request.js拦截器是否正确添加 |
| 接口返回403 | 非管理员用户访问 | 确认数据库中用户role为1 |
