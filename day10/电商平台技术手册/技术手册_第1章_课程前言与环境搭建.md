## 1.1 课程前言描述（15分钟）

### 项目背景：电商平台的行业现状

"同学们，欢迎来到基于SpringBoot AI花之恋鲜花电商平台实训课程。在开始敲代码之前，我们先来了解一下——我们为什么要做这个项目？"

"2024年，中国网络零售额已经超过15万亿元，电商早已不是什么新鲜事物，而是每个企业的基础设施。淘宝、京东、拼多多——这些App每天处理数十亿笔交易。而支撑这些交易的，就是电商平台系统。"

"我们这个项目，就是要做一个'麻雀虽小五脏俱全'的鲜花电商平台——花之恋。它有前台商城——用户可以浏览鲜花商品、看详情、下单购买；也有后台管理——管理员可以管理商品、处理订单、查看数据统计。"

### 当前企业技术应用范围

"从技术角度看，企业级Java Web项目几乎都用SpringBoot。为什么？三个原因——"

**（板书）**

```
SpringBoot 三大核心优势：

1. 自动配置 —— 不用写一大堆XML，引入依赖就能用
2. 起步依赖 —— 一个starter搞定一组jar包
3. 内嵌Tomcat —— 打成jar包直接运行，不用部署到外部服务器
```

"我们这个项目，后端用SpringBoot 3.5.3提供REST API，前端商城用Vue3+ElementPlus构建PC端页面，后台管理用Bootstrap5构建PC端页面。这就是企业里最常见的前后端分离架构。"

## 1.2 开发环境搭建（60分钟）

### 步骤1：安装JDK 17

"首先安装JDK。我们项目使用SpringBoot 3.x，它要求JDK最低版本是17，不再支持JDK 8。这一点和以前的旧项目不同，大家一定要注意。"

**操作：**

1. 访问 Oracle JDK 17 下载页：https://www.oracle.com/java/technologies/javase/jdk17-archive-downloads.html
2. 下载 `jdk-17_windows-x64_bin.exe`（需要Oracle账号，也可以搜索"JDK17网盘"找国内镜像）
3. 双击安装包，一路Next，安装路径保持默认 `C:\Program Files\Java\jdk-17`

**配置环境变量：**

1. 右键"此电脑"→"属性"→"高级系统设置"→"环境变量"
2. 在"系统变量"中点击"新建"：
   - 变量名：`JAVA_HOME`
   - 变量值：`C:\Program Files\Java\jdk-17`
3. 找到系统变量中的 `Path`，点击"编辑"→"新建"，添加：
   ```
   %JAVA_HOME%\bin
   ```
4. 点击"确定"保存

**验证安装：**

打开命令提示符（Win+R 输入 cmd），输入：

```bash
java -version
```

**预期效果：**

```
java version "17.0.x" 2024-xx-xx LTS
Java(TM) SE Runtime Environment (build 17.0.x+x-x)
Java HotSpot(TM) 64-Bit Server VM (build 17.0.x+x-x, mixed mode, sharing)
```

再输入：

```bash
javac -version
```

**预期效果：**

```
javac 17.0.x
```

> ⚠️ 如果提示"java不是内部命令"，检查JAVA_HOME路径是否正确，Path中是否添加了%JAVA_HOME%\bin。如果之前安装过JDK 8，需要将JDK 17的Path条目移到最上面，或者删除旧的JDK 8相关配置。

### 步骤2：安装IDEA 2024.3+

"Java开发最主流的IDE就是IntelliJ IDEA。我们使用2024.3或更高版本，这个版本对JDK 17和SpringBoot 3.x有很好的支持。"

**操作：**

1. 访问 https://www.jetbrains.com/idea/download/ 下载 IntelliJ IDEA（Ultimate版或Community版均可，Ultimate版有30天试用）
2. 双击安装包，Next → 勾选"Add launchers dir to the PATH" → 勾选".java" → Install
3. 安装完成后打开IDEA

**IDEA基本配置：**

1. **设置编码为UTF-8：**
   - File → Settings → Editor → File Encodings
   - 将 Global Encoding、Project Encoding、Default encoding for properties files 都设为 `UTF-8`

2. **配置JDK 17：**
   - File → Project Structure → Project
   - SDK选择 `17`（如果列表中没有，点击"Add SDK"→"JDK"，选择 `C:\Program Files\Java\jdk-17`）
   - Language level 选择 `17`

3. **配置Maven：**
   - File → Settings → Build, Execution, Deployment → Build Tools → Maven
   - Maven home directory：选择 `Bundled (Maven 3)` 或你本地安装的Maven路径
   - User settings file：点击 Override，指定你的 Maven settings.xml 路径

4. **配置Maven镜像（加速下载）：**

   找到Maven的settings.xml文件（通常在 `C:\Users\你的用户名\.m2\settings.xml`），如果不存在就创建。写入以下内容：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<settings xmlns="http://maven.apache.org/SETTINGS/1.0.0"
          xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
          xsi:schemaLocation="http://maven.apache.org/SETTINGS/1.0.0
          http://maven.apache.org/xsd/settings-1.0.0.xsd">
    <mirrors>
        <mirror>
            <id>aliyun</id>
            <mirrorOf>central</mirrorOf>
            <name>Aliyun Maven</name>
            <url>https://maven.aliyun.com/repository/public</url>
        </mirror>
    </mirrors>
</settings>
```

## 1.4 利用AI生成项目需求（30分钟）


## 1.5 项目数据库设计与初始化（30分钟）

"需求理清了，下一步是设计数据库。数据库设计是整个项目的地基，地基不稳，大厦将倾。"

"我们的数据库表结构已经在统一技术规范中定义好了，下面严格按照规范来创建。"

### 数据库表设计说明

"我们先来看看3张表的设计思路——"

**用户表（user）：** 存储用户信息，包括普通用户和管理员，通过role字段区分。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键，自增 |
| username | VARCHAR(50) | 用户名，唯一 |
| password | VARCHAR(64) | 密码，MD5加密存储 |
| phone | VARCHAR(20) | 手机号 |
| role | INT | 角色：0普通用户，1管理员 |
| status | INT | 状态：0禁用，1正常 |
| create_time | DATETIME | 创建时间 |

**商品表（product）：** 存储商品信息，包含价格、库存、销量和上下架状态。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键，自增 |
| name | VARCHAR(100) | 商品名称 |
| price | DECIMAL(10,2) | 价格 |
| original_price | DECIMAL(10,2) | 原价 |
| description | TEXT | 商品描述 |
| category | VARCHAR(50) | 分类：鲜花/永生花/花束/花篮/绿植/礼品花 |
| image | VARCHAR(500) | 商品图片URL |
| stock | INT | 库存 |
| sales | INT | 销量 |
| status | INT | 状态：0下架，1上架 |
| create_time | DATETIME | 创建时间 |

**订单表（orders）：** 存储订单信息，用orders避免与MySQL关键字ORDER冲突。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键，自增 |
| order_no | VARCHAR(64) | 订单号，唯一 |
| user_id | INT | 用户ID |
| product_id | INT | 商品ID |
| product_name | VARCHAR(100) | 商品名称（冗余） |
| product_image | VARCHAR(500) | 商品图片（冗余） |
| product_price | DECIMAL(10,2) | 商品单价（冗余） |
| quantity | INT | 购买数量 |
| total_price | DECIMAL(10,2) | 订单总价 |
| receiver_name | VARCHAR(50) | 收货人姓名 |
| receiver_phone | VARCHAR(20) | 收货人手机号 |
| receiver_address | VARCHAR(255) | 收货地址 |
| status | INT | 订单状态：0待付款 1已付款 2已发货 3已完成 4已取消 |
| create_time | DATETIME | 创建时间 |
| update_time | DATETIME | 更新时间 |

"注意几个关键点——"
1. 表名统一使用小写英文，不加前缀 `t_`
2. 订单表用 `orders` 避免与MySQL关键字 `ORDER` 冲突
3. 订单表冗余存储商品名称、图片、价格，因为商品价格可能变化，订单需要保存下单时的价格
4. 订单表使用 `receiver_name`、`receiver_phone`、`receiver_address` 存储收货人信息

## 本章常见问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| java -version 显示1.8而非17 | 系统安装了多个JDK，Path中旧版本优先 | 将JDK 17的Path条目移到最上面，或删除旧JDK的Path条目 |
| IDEA创建项目时找不到JDK 17 | IDEA未识别到JDK 17安装路径 | File → Project Structure → Add SDK → JDK → 选择 `C:\Program Files\Java\jdk-17` |
| Spring Initializr中找不到3.5.3版本 | 网络问题或IDEA缓存 | 检查网络连接，或手动修改pom.xml中的版本号 |
| IDEA中Maven下载依赖很慢 | 使用的是Maven中央仓库（国外） | 配置阿里云Maven镜像，见步骤2 |
| npm install 很慢 | npm默认使用国外registry | 执行 npm config set registry https://registry.npmmirror.com |
| MySQL连接失败 | MySQL服务未启动 | 打开服务管理器（services.msc），找到MySQL80服务，右键启动 |
| SpringBoot启动报驱动类错误 | 使用了旧的MySQL驱动类名 | 确认application.yml中driver-class-name为 `com.mysql.cj.jdbc.Driver` |
| 创建orders表报错 | orders可能与MySQL保留字冲突 | 表名和列名都用反引号包裹 `` `orders` `` |
| picsum.photos图片加载不出来 | 网络问题 | 可替换为其他占位图服务，如 placeholder.com |
| VSCode中Vue3没有语法高亮 | 安装了Vetur（Vue2插件）而非Vue - Official | 卸载Vetur，安装"Vue - Official"插件 |
