## 14.1 AI生成三层架构代码

> "假设老板突然说'加一个鲜花分类管理功能'，你需要新建一张category表，然后写Entity、Mapper、Service、Controller……一套下来少说半小时。但如果用AI，5分钟就能搞定！"

### 14.1.1 AI代码生成原理

```
输入：建表SQL + 技术栈信息
  ↓
AI理解：表名、字段名、字段类型、主键
  ↓
生成代码：
  - Entity类（字段对应列，类型映射）
  - Mapper接口 + XML（CRUD方法 + SQL）
  - Service类（业务逻辑）
  - Controller类（REST接口）
  ↓
复制到项目 → 审查微调 → 验证！
```

### 14.1.2 准备演示：创建category分类表

连接到 `ecommerce` 数据库，执行以下建表SQL：

```sql
-- 商品分类表（花之恋鲜花电商）

    id INT PRIMARY KEY AUTO_INCREMENT COMMENT '分类ID',
```

### 14.1.3 用AI生成Entity类

**给AI的Prompt：**

```
根据以下MySQL建表语句，生成SpringBoot 3.x项目的Entity类。

建表SQL：
CREATE TABLE category (
    id INT PRIMARY KEY AUTO_INCREMENT COMMENT '分类ID',
    name VARCHAR(50) NOT NULL COMMENT '分类名称',
    sort_order INT DEFAULT 0 COMMENT '排序号，越小越靠前',
    status INT DEFAULT 1 COMMENT '状态：1启用 0禁用',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='鲜花商品分类表';

要求：
1. 包名：com.ecommerce.entity
2. 使用Lombok注解（@Data, @NoArgsConstructor, @AllArgsConstructor）
3. 字段类型用Java包装类（Integer, String等）
4. 日期类型用java.time.LocalDateTime（JDK 17+）
5. 添加中文注释
6. 兼容SpringBoot 3.x + JDK 17
```

**AI生成的Entity类：**

创建文件 `backend/src/main/java/com/ecommerce/entity/Category.java`：

```java
package com.ecommerce.entity;

import lombok.AllArgsConstructor;
import lombok.Data;
import lombok.NoArgsConstructor;

import java.time.LocalDateTime;

/**
 * 鲜花商品分类实体类
 */
@Data
@NoArgsConstructor
@AllArgsConstructor
public class Category {

    /** 分类ID */
    private Integer id;

    /** 分类名称 */
    private String name;

    /** 排序号，越小越靠前 */
    private Integer sortOrder;

    /** 状态：1启用 0禁用 */
    private Integer status;

    /** 创建时间 */
    private LocalDateTime createTime;

    /** 更新时间 */
    private LocalDateTime updateTime;
}
```

> "注意：使用 `java.time.LocalDateTime` 而不是 `java.util.Date`，这是JDK 17+的推荐做法。MyBatis配置了 `map-underscore-to-camel-case: true` 后会自动将 `sort_order` 映射为 `sortOrder`。"

### 14.1.4 用AI生成Mapper接口和XML

**给AI的Prompt：**

```
根据以下Entity类，生成MyBatis的Mapper接口和对应的XML映射文件。

Entity类：com.ecommerce.entity.Category（字段：id, name, sortOrder, status, createTime, updateTime）
表名：category

要求：
1. 包名：com.ecommerce.mapper
2. 方法列表：
   - List<Category> selectAll() 查询所有分类（按sort_order升序）
   - Category selectById(Integer id) 根据ID查询
   - int insert(Category category) 新增分类
   - int updateById(Category category) 更新分类（动态SQL）
   - int deleteById(Integer id) 根据ID删除
3. XML文件namespace对应Mapper接口全限定名
4. 使用#{param}占位符（防SQL注入）
5. 兼容MyBatis SpringBoot Starter 3.0.4
```

**AI生成的Mapper接口：**

创建文件 `backend/src/main/java/com/ecommerce/mapper/CategoryMapper.java`：

```java
package com.ecommerce.mapper;

import com.ecommerce.entity.Category;
import org.apache.ibatis.annotations.Param;

import java.util.List;

/**
 * 鲜花商品分类Mapper接口
 */
public interface CategoryMapper {

    List<Category> selectAll();

    Category selectById(@Param("id") Integer id);

    int insert(Category category);

    int updateById(Category category);

    int deleteById(@Param("id") Integer id);
}
```

**AI生成的Mapper XML：**

创建文件 `src/main/resources/mapper/CategoryMapper.xml`：

```xml
<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">
<mapper namespace="com.ecommerce.mapper.CategoryMapper">

    <select id="selectAll" resultType="com.ecommerce.entity.Category">
        SELECT id, name, sort_order, status, create_time, update_time
        FROM category
        ORDER BY sort_order ASC
    </select>

    <select id="selectById" resultType="com.ecommerce.entity.Category">
        SELECT id, name, sort_order, status, create_time, update_time
        FROM category
        WHERE id = #{id}
    </select>

    <insert id="insert" useGeneratedKeys="true" keyProperty="id">
        INSERT INTO category (name, sort_order, status)
        VALUES (#{name}, #{sortOrder}, #{status})
    </insert>

    <update id="updateById">
        UPDATE category
        <set>
            <if test="name != null">name = #{name},</if>
            <if test="sortOrder != null">sort_order = #{sortOrder},</if>
            <if test="status != null">status = #{status},</if>
        </set>
        WHERE id = #{id}
    </update>

    <delete id="deleteById">
        DELETE FROM category WHERE id = #{id}
    </delete>

</mapper>
```

### 14.1.5 用AI生成Service类

**给AI的Prompt：**

```
根据以下Mapper接口，生成对应的Service类。

Mapper接口方法：selectAll, selectById, insert, updateById, deleteById

要求：
1. 包名：com.ecommerce.service
2. 加@Service注解
3. 注入CategoryMapper，用@Autowired
4. 方法直接调用Mapper对应方法
5. 添加中文注释
```

**AI生成的Service类：**

创建文件 `backend/src/main/java/com/ecommerce/service/CategoryService.java`：

```java
package com.ecommerce.service;

import com.ecommerce.entity.Category;
import com.ecommerce.mapper.CategoryMapper;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;

import java.util.List;

/**
 * 鲜花商品分类Service
 */
@Service
public class CategoryService {

    @Autowired
    private CategoryMapper categoryMapper;

    public List<Category> findAll() {
        return categoryMapper.selectAll();
    }

    public Category findById(Integer id) {
        return categoryMapper.selectById(id);
    }

    public int insert(Category category) {
        return categoryMapper.insert(category);
    }

    public int updateById(Category category) {
        return categoryMapper.updateById(category);
    }

    public int deleteById(Integer id) {
        return categoryMapper.deleteById(id);
    }
}
```

### 14.1.6 用AI生成Controller类

**给AI的Prompt：**

```
根据以下Service类，生成对应的REST Controller类。

Service方法：findAll, findById, insert, updateById, deleteById

要求：
1. 包名：com.ecommerce.controller
2. 加@RestController和@RequestMapping("/api/category")
3. 注入CategoryService，用@Autowired
4. 使用com.ecommerce.util.Result作为统一返回格式
5. 接口列表：
   - GET /api/category/list 查询所有
   - GET /api/category/{id} 根据ID查询
   - POST /api/category/add 新增（@RequestBody）
   - PUT /api/category/update 更新（@RequestBody）
   - DELETE /api/category/{id} 删除
6. 兼容SpringBoot 3.x + jakarta.servlet
7. 添加中文注释
```

**AI生成的Controller类：**

创建文件 `backend/src/main/java/com/ecommerce/controller/CategoryController.java`：

```java
package com.ecommerce.controller;

import com.ecommerce.entity.Category;
import com.ecommerce.service.CategoryService;
import com.ecommerce.util.Result;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.web.bind.annotation.*;

import java.util.List;

/**
 * 鲜花商品分类Controller
 */
@RestController
@RequestMapping("/api/category")
public class CategoryController {

    @Autowired
    private CategoryService categoryService;

    /**
     * 查询所有分类
     * GET /api/category/list
     */
    @GetMapping("/list")
    public Result<List<Category>> list() {
        return Result.success(categoryService.findAll());
    }

    /**
     * 根据ID查询分类
     * GET /api/category/{id}
     */
    @GetMapping("/{id}")
    public Result<Category> getById(@PathVariable Integer id) {
        var category = categoryService.findById(id);
        if (category != null) {
            return Result.success(category);
        }
        return Result.error("分类不存在");
    }

    /**
     * 新增分类
     * POST /api/category/add
     */
    @PostMapping("/add")
    public Result<String> add(@RequestBody Category category) {
        categoryService.insert(category);
        return Result.success("新增成功");
    }

    /**
     * 更新分类
     * PUT /api/category/update
     */
    @PutMapping("/update")
    public Result<String> update(@RequestBody Category category) {
        categoryService.updateById(category);
        return Result.success("更新成功");
    }

    /**
     * 根据ID删除分类
     * DELETE /api/category/{id}
     */
    @DeleteMapping("/{id}")
    public Result<String> delete(@PathVariable Integer id) {
        categoryService.deleteById(id);
        return Result.success("删除成功");
    }
}
```

> "注意：Controller使用 `com.ecommerce.util.Result` 作为统一返回格式，和项目其他Controller保持一致。使用 `@PutMapping` 和 `@DeleteMapping` 符合RESTful规范。使用了JDK 17的 `var` 关键字。"

### 14.1.8 代码审查要点

| 审查项 | 常见问题 | 调整方法 |
|--------|---------|---------|
| 字段映射 | 数据库下划线命名 vs Java驼峰命名 | 确认MyBatis配置了mapUnderscoreToCamelCase |
| 日期类型 | 使用Date还是LocalDateTime | JDK 17+推荐用LocalDateTime |
| 参数校验 | AI通常不会添加参数校验 | 手动加@NotNull、@Size等注解 |
| 返回格式 | AI可能用Map返回 | 统一使用Result类 |
| SQL注入 | 使用#{}而不是${} | 确认XML中全部用#{}占位符 |
| Servlet版本 | javax还是jakarta | SpringBoot 3.x必须用jakarta |

## 14.4 项目打包部署

> "功能检查全部通过后，来学习如何打包和部署项目。注意：项目运行需要JDK 17环境。"

### 14.4.1 后端打包部署

#### 步骤1：确认JDK 17环境

```bash
java -version
# 应该显示 openjdk version "17.x.x" 或更高
```

> "SpringBoot 3.x + Spring AI要求JDK 17+，JDK 8或11无法运行本项目。"

#### 步骤2：执行Maven打包

```bash
mvn clean package -DskipTests
```

> "`-DskipTests` 表示跳过测试，因为打包时可能没有数据库连接。"

**预期效果：** 在 `target/` 目录下生成 `ecommerce-0.0.1-SNAPSHOT.jar` 文件。

#### 步骤3：运行jar包

```bash
java -jar target/ecommerce-0.0.1-SNAPSHOT.jar
```

如果需要覆盖配置（如生产环境数据库密码）：

```bash
java -jar target/ecommerce-0.0.1-SNAPSHOT.jar \
  --spring.datasource.password=生产密码 \
  --server.port=8081
```

**预期效果：** SpringBoot项目正常启动，访问 `http://localhost:8080` 能正常使用。

#### 步骤4：后台运行（Linux服务器）

```bash
# 后台运行，日志输出到文件
nohup java -jar ecommerce-0.0.1-SNAPSHOT.jar > app.log 2>&1 &

# 查看进程
ps aux | grep java

# 停止服务
kill -15 <PID>
```

### 14.4.2 前端打包部署

#### 步骤1：配置前端API地址

确保前端的axios请求指向正确的后端地址。如果前后端部署在同一台服务器，用相对路径 `/api/` 即可。

#### 步骤2：执行前端打包

```bash
cd frontend
npm run build
```

**预期效果：** 在 `dist/` 目录下生成打包后的静态文件。

#### 步骤3：部署前端

**方式一：放入SpringBoot静态资源目录**

将 `dist/` 目录的文件复制到 `src/main/resources/static/` 下，随jar包一起部署。

**方式二：使用Nginx部署**

Nginx配置示例：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    location / {
        root /usr/share/nginx/html/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # API代理到后端
    location /api/ {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## 常见问题

| 序号 | 问题 | 原因 | 解决方案 |
|------|------|------|---------|
| 1 | AI生成的Entity字段类型不对 | AI对MySQL类型映射不准确 | 手动调整：INT→Integer, VARCHAR→String, DATETIME→LocalDateTime |
| 2 | AI生成的Mapper XML报错 | namespace路径不对 | 检查namespace是否和Mapper接口全限定名一致 |
| 3 | AI生成的代码用javax.servlet | SpringBoot 3.x需要jakarta | 替换import javax.servlet为import jakarta.servlet |
| 4 | Maven打包失败 | 测试用例报错或依赖冲突 | 用`-DskipTests`跳过测试，检查依赖版本 |
| 5 | 前端打包后页面空白 | 路由模式或资源路径问题 | 检查Vue Router的history模式，Nginx配置try_files |
| 6 | jar包运行报JDK版本错误 | 运行环境JDK版本低于17 | 安装JDK 17+，确认java -version显示17+ |
| 7 | 数据库连不上 | 生产环境配置不对 | 用命令行参数覆盖：`--spring.datasource.url=xxx` |
| 8 | 答辩时AI功能演示失败 | 网络问题或API Key过期 | 提前测试，准备Mock数据备用方案 |
