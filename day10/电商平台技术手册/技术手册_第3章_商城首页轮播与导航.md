> **学习目标：** 完成商城布局页ShopLayout和首页的轮播图、分类导航的实现
> **前置条件：** 完成第2章的Vue3+ElementPlus项目初始化
> **本章产出：** ShopLayout商城布局页 + Home首页（轮播图+分类导航）

## 3.1 商城首页整体布局（15分钟）

"同学们，从今天开始，我们正式进入页面开发阶段。商城首页是用户看到的第一个页面，它决定了用户对整个平台的第一印象。"

"和移动端不同，PC端电商有一个重要概念——**布局页（Layout）**。搜索栏和导航栏不属于某一个页面，而是所有商城页面共享的'外壳'。所以我们先搭建ShopLayout.vue，再在它的'内容区'里填充Home.vue。"

**（板书）**

"为什么要拆出ShopLayout？因为搜索栏、导航栏、底部版权是所有商城页面共享的。用户从首页点进鲜花分类页，导航栏不应该消失。ShopLayout就是这些公共元素的容器，而 `<router-view />` 就是页面内容的'插槽'。"

"PC端和移动端的另一个大区别——**没有底部TabBar**。PC端用顶部导航栏代替底部TabBar，更符合桌面用户的操作习惯。"

## 3.3 实现首页轮播图（45分钟）

### 步骤1：轮播图设计思路

"PC端的轮播图和移动端完全不同。移动端是简单的图片轮播，但PC端我们需要更有视觉冲击力的设计——**渐变背景 + 大字标题 + emoji**。"

"为什么不用图片？因为——"
1. 图片需要设计资源，教学阶段不方便准备
2. 纯色渐变 + 文字的方案加载速度更快
3. 通过CSS渐变可以轻松实现浪漫的鲜花主题效果

### 步骤2：搜索功能的数据流

"搜索功能涉及ShopLayout和Home两个组件的协作。完整的搜索流程——"

```
用户操作               组件                    路由
─────────────────────────────────────────────────────────
1. 输入"玫瑰"        ShopLayout.searchQuery
2. 按回车            ShopLayout.handleSearch()
3. 路由跳转                                 /shop?query=玫瑰
4. Home监听query     Home watch(route.query.query)
5. 执行搜索逻辑      → ElMessage提示
```

"Home.vue通过 `watch(() => route.query.query, ...)` 监听URL中的query参数。当ShopLayout的搜索框触发跳转后，Home就能感知到并执行搜索。这种设计叫**URL驱动**——URL是两个组件之间的'信使'。"

"分类导航也是同样的原理——点击分类后跳转到 `/shop?category=鲜花`，Home.vue通过 `watch(route.query.category)` 接收。"

## 本章常见问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 轮播图不自动播放 | 没有设置interval属性 | 检查el-carousel上是否设置了`:interval="4000"` |
| 轮播图箭头不显示 | arrow属性设为never | PC端应设为`arrow="hover"`，鼠标悬停时显示 |
| 分类图标不显示 | categories数据未定义 | 检查categories数组是否在script setup中定义 |
| 搜索不跳转 | ShopLayout的handleSearch未绑定 | 检查el-input是否绑定了@keyup.enter |
| Home收不到搜索词 | watch未监听route.query.query | 检查watch第一个参数是否是`() => route.query.query` |
| 页面没有导航栏 | 路由未配置ShopLayout | 检查router中/shop是否使用了ShopLayout作为父组件 |
| Logo点击不跳转 | @click事件未绑定 | 检查Logo元素是否有`@click="$router.push('/shop')"` |
