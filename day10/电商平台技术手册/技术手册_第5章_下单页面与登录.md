"ElementPlus表单验证和Vant4弹窗验证的关键区别——"

| 对比项 | Vant4（旧版） | ElementPlus（新版） |
|--------|--------------|---------------------|
| 验证方式 | `van-dialog` 的 `beforeClose` 回调 | `el-form` 的 `rules` + `validate()` |
| 验证失败不关闭弹窗 | `done(false)` | 表单验证失败不通过，不弹出确认框 |
| 手机号验证 | 手动正则判断 | rules中配置`pattern: /^1[3-9]\d{9}$/` |
| 表单引用 | 无 | `ref="orderFormRef"` + `this.$refs.orderFormRef.validate()` |

### 步骤1：完善路由守卫

"我们在第2章已经配置了路由守卫的基本框架，现在来确保它工作正常。"

确认 `E:\HuaShan\ecommerce\frontend\src\router\index.js` 中的路由守卫代码：

```javascript
// 路由守卫：需要登录的页面，未登录则跳转到登录页
router.beforeEach((to, from, next) => {
  // 设置页面标题
  document.title = to.meta.title || '花之恋鲜花电商'

  // 检查是否需要登录
  if (to.meta.requiresAuth) {
    const token = localStorage.getItem('token')
    if (!token) {
      // 未登录，跳转到登录页，并记录原目标路径
      next({
        path: '/login',
        query: { redirect: to.fullPath }
      })
    } else {
      next()
    }
  } else {
    next()
  }
})
```

"路由守卫的工作流程——"

**（板书）**

```
用户访问需要登录的页面（如 /order/1）
  │

  │
  ├─ Token存在 → next() 放行
  │
  └─ Token不存在 → 跳转到 /login?redirect=/order/1
                      │
                      ↓
                   🌸 花之恋 登录页
                      │
                      ↓
                   登录成功
                      │
```
