## 9.1 ProductMapper补充方法

"管理后台需要对商品进行增删改查操作。之前我们只有查询和库存相关的方法，现在要补充新增、修改、逻辑删除和状态修改的方法。为什么要用逻辑删除？因为删除商品后，历史订单中引用的商品信息还需要保留，如果物理删除，数据就找不回来了。"

### 9.1.1 更新ProductMapper接口

在 `ProductMapper.java` 中补充以下方法：

```java
package com.ecommerce.mapper;

import com.ecommerce.entity.Product;
import org.apache.ibatis.annotations.Mapper;
import org.apache.ibatis.annotations.Param;

import java.util.List;

@Mapper
public interface ProductMapper {

    /**
     * 查询所有商品
     */
    List<Product> findAll();

    /**
     * 分页查询商品列表（只查上架商品，前台使用）
     */
    List<Product> findByPage(int offset, int size);

    /**
     * 根据ID查询商品
     */

    /**
     * 查询上架商品总数
     */
    long count();

    /**
     * 根据关键词搜索商品（LIKE模糊查询，第13章升级为FULLTEXT）
     */
    List<Product> searchByKeyword(String keyword);

    /**
     * 扣减库存
     */
    int decreaseStock(@Param("id") Integer id, @Param("quantity") Integer quantity);

    /**
     * 增加销量
     */
    int increaseSales(@Param("id") Integer id, @Param("quantity") Integer quantity);

    /**
     * 新增商品
     */
    int insert(Product product);

    /**
     * 修改商品（动态更新非null字段）
     */
    int update(Product product);

     * 修改商品状态（上下架/逻辑删除）
     * 逻辑删除：将status设为0
     */
    int updateStatus(@Param("id") Integer id, @Param("status") Integer status);

    /**
     * 分页查询所有商品（不过滤status，管理后台使用）
     */
    List<Product> findAllByPage(@Param("offset") int offset, @Param("size") int size);

    /**
     * 查询所有商品总数（不过滤status，管理后台使用）
```

## 9.2 ProductService补充方法

"Service层封装业务逻辑。管理接口的业务相对简单，主要是调用Mapper方法。但我们要注意：新增商品时需要设置默认值，逻辑删除时不能真的删除数据。"

在 `ProductService.java` 中补充以下方法：

```java
package com.ecommerce.service;

import com.ecommerce.entity.PageResult;
import com.ecommerce.entity.Product;
import com.ecommerce.mapper.ProductMapper;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;

import java.util.Date;
import java.util.List;

/**
 * 商品服务类
 */
@Service
public class ProductService {

    @Autowired
    private ProductMapper productMapper;

    /**
     * 分页查询商品列表（前台，只查上架商品）
     */
    public PageResult<Product> getProductList(int page, int size) {
        if (page < 1) {
            page = 1;
        }
        if (size < 1) {
            size = 10;
        }
        int offset = (page - 1) * size;
        List<Product> list = productMapper.findByPage(offset, size);
        long total = productMapper.count();
        return new PageResult<>(page, size, total, list);
    }

    /**
     * 查询商品详情
     */
    public Product getProductById(Integer id) {
        if (id == null) {
            return null;
        }
        return productMapper.findById(id);
    }

    /**
     * 搜索商品（LIKE模糊查询，第13章升级为FULLTEXT全文索引）
     */
    public List<Product> searchProduct(String keyword) {
        if (keyword == null || keyword.trim().isEmpty()) {
            return productMapper.findByPage(0, 20);
        }
        return productMapper.searchByKeyword(keyword.trim());
    }

    /**
     * 按分类分页查询商品（第11章补充）
     */
    public PageResult<Product> getProductListByCategory(int page, int size, String category) {
        if (page < 1) {
            page = 1;
        }
        if (size < 1) {
            size = 10;
        }
        int offset = (page - 1) * size;
        List<Product> list = productMapper.findByCategoryPage(offset, size, category);
        long total = productMapper.countByCategory(category);
        return new PageResult<>(page, size, total, list);
    }

    /**
     * 全文索引搜索商品（第13章补充）
     */
    public List<Product> searchProductFulltext(String keyword) {
        if (keyword == null || keyword.trim().isEmpty()) {
            return productMapper.findByPage(0, 20);
        }
        return productMapper.searchByKeywordFulltext(keyword.trim());
    }

    /**
     * 新增商品
     *
     * @param product 商品信息
     * @return 操作结果消息
     */
    public String addProduct(Product product) {
        // 设置默认值
        if (product.getStatus() == null) {
            product.setStatus(1);        // 默认上架
        }
        if (product.getStock() == null) {
            product.setStock(0);         // 默认库存为0
        }
        if (product.getSales() == null) {
            product.setSales(0);         // 默认销量为0
        }
        product.setCreateTime(new Date());

        int rows = productMapper.insert(product);
        if (rows > 0) {
            return "新增商品成功";
        }
        return "新增商品失败";
    }

    /**
     * 修改商品
     *
     * @param product 商品信息（必须包含id）
     * @return 操作结果消息
     */
    public String updateProduct(Product product) {
        // 检查商品是否存在
        Product existProduct = productMapper.findById(product.getId());
        if (existProduct == null) {
            return "商品不存在";
        }

        int rows = productMapper.update(product);
        if (rows > 0) {
            return "修改商品成功";
        }
        return "修改商品失败";
    }

    /**
     * 修改商品状态/上下架
     * 注意：逻辑删除也使用此方法，将status设为0
     *
     * @param id     商品ID
     * @param status 状态：0-下架/删除，1-上架
     * @return 操作结果消息
     */
    public String updateProductStatus(Integer id, Integer status) {
        // 参数校验
        if (status == null || (status != 0 && status != 1)) {
            return "状态值无效，0-下架，1-上架";
        }

        // 检查商品是否存在
        Product existProduct = productMapper.findById(id);
        if (existProduct == null) {
            return "商品不存在";
        }

        int rows = productMapper.updateStatus(id, status);
        if (rows > 0) {
            return status == 1 ? "商品已上架" : "商品已下架";
        }
        return "修改状态失败";
    }
}
```

**预期效果：** ProductService管理方法：addProduct设置默认值后插入、updateProduct先检查存在再修改、updateProductStatus校验状态值后修改（逻辑删除也使用此方法，将status设为0）。所有管理方法返回String类型的结果消息。getProductList使用findByPage+count（前台分页），管理后台列表使用findAllByPage+countAll。

## 9.3 管理员权限验证

"管理接口只有管理员才能访问。我们怎么判断当前用户是不是管理员？还记得User表有个role字段吗？0是普通用户，1是管理员。但我们的JWT Token中只存了userId和username，没有存role。所以我们需要先根据userId查询用户信息，再判断role。"

"为了方便，我们在Controller中编写一个checkAdmin私有方法，在需要权限校验的接口中直接调用。"

checkAdmin方法的实现逻辑：

```java
/**
 * 验证管理员权限
 * 从拦截器已设置的request属性获取userId，查user表确认role==1
 *
 * @param request HTTP请求对象
 * @return null表示验证通过，非null表示验证失败的Result
 */
private Result<?> checkAdmin(HttpServletRequest request) {
    // 第一步：从request属性获取userId（由拦截器设置）
    Integer userId = (Integer) request.getAttribute("userId");
    if (userId == null) {
        return Result.error(401, "未登录");
    }

    // 第二步：查询用户信息
    User user = userService.getUserById(userId);
    if (user == null) {
        return Result.error(401, "用户不存在");
    }

    // 第三步：检查角色（role==1为管理员）
    if (user.getRole() == null || user.getRole() != 1) {
        return Result.error(403, "无权限");
    }

    // 验证通过
    return null;
}
```

**预期效果：** 调用checkAdmin方法，传入request，如果用户是管理员返回null（验证通过），否则返回对应的错误Result（401未登录或403无权限）。Controller中使用方式：

```java
Result<?> authResult = checkAdmin(request);
if (authResult != null) {
    return (Result) authResult;
}
```

## 9.4 AdminProductController编写

"商品管理控制器放在/admin/product路径下，和前台的商品查询接口区分开。每个接口都需要先验证管理员权限。"

在 `com.ecommerce.controller` 包下创建 `AdminProductController.java`：

```java
package com.ecommerce.controller;

import com.ecommerce.entity.PageResult;
import com.ecommerce.entity.Product;
import com.ecommerce.entity.Result;
import com.ecommerce.entity.User;
import com.ecommerce.mapper.ProductMapper;
import com.ecommerce.service.ProductService;
import com.ecommerce.service.UserService;
import jakarta.servlet.http.HttpServletRequest;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.web.bind.annotation.*;

import java.util.List;
import java.util.Map;

/**
 * 管理员-商品管理控制器
 */
@RestController
@RequestMapping("/api/admin/product")
public class AdminProductController {

    @Autowired
    private ProductService productService;

    @Autowired
    private UserService userService;

    @Autowired
    private ProductMapper productMapper;

    /**
     * 验证管理员权限
     */
    private Result<?> checkAdmin(HttpServletRequest request) {
        Integer userId = (Integer) request.getAttribute("userId");
        if (userId == null) {
            return Result.error(401, "未登录");
        }
        User user = userService.getUserById(userId);
        if (user == null) {
            return Result.error(401, "用户不存在");
        }
        if (user.getRole() == null || user.getRole() != 1) {
            return Result.error(403, "无权限");
        }
        return null;
    }

    /**
     * 查询商品列表（分页，不过滤status）
     * GET /api/admin/product/list?page=1&size=10
     * 需要管理员权限
     */
    @GetMapping("/list")
    public Result<PageResult<Product>> list(
            @RequestParam(defaultValue = "1") int page,
            @RequestParam(defaultValue = "10") int size,
            HttpServletRequest request) {
        // 第一步：验证管理员权限
        Result<?> authResult = checkAdmin(request);
        if (authResult != null) {
            return (Result) authResult;
        }

        // 第二步：使用productMapper.findAllByPage()查询所有商品（不过滤status）
        int offset = (page - 1) * size;
        List<Product> list = productMapper.findAllByPage(offset, size);
        long total = productMapper.countAll();
        PageResult<Product> pageResult = new PageResult<>(page, size, total, list);
        return Result.success("查询成功", pageResult);
    }

    /**
     * 新增商品
     * POST /api/admin/product/add
     * 需要管理员权限
     *
     * @param product 商品信息
     * @param request HTTP请求对象
     * @return 操作结果
     */
    @PostMapping("/add")
    public Result<String> addProduct(@RequestBody Product product,
                                      HttpServletRequest request) {
        // 第一步：验证管理员权限
        Result<?> authResult = checkAdmin(request);
        if (authResult != null) {
            return (Result) authResult;
        }

        // 第二步：参数校验
        if (product.getName() == null || product.getName().trim().isEmpty()) {
            return Result.error(400, "商品名称不能为空");
        }
        if (product.getPrice() == null) {
            return Result.error(400, "商品价格不能为空");
        }

        // 第三步：调用Service新增商品
        String result = productService.addProduct(product);
        if (result.contains("成功")) {
            return Result.success(result, result);
        } else {
            return Result.error(400, result);
        }
    }

    /**
     * 修改商品
     * PUT /api/admin/product/update
     * 需要管理员权限
     *
     * @param product 商品信息（必须包含id）
     * @param request HTTP请求对象
     * @return 操作结果
     */
    @PutMapping("/update")
    public Result<String> updateProduct(@RequestBody Product product,
                                         HttpServletRequest request) {
        // 第一步：验证管理员权限
        Result<?> authResult = checkAdmin(request);
        if (authResult != null) {
            return (Result) authResult;
        }

        // 第二步：参数校验
        if (product.getId() == null) {
            return Result.error(400, "商品ID不能为空");
        }

        // 第三步：调用Service修改商品
        String result = productService.updateProduct(product);
        if (result.contains("成功")) {
            return Result.success(result, result);
        } else {
            return Result.error(400, result);
        }
    }

    /**
     * 删除商品（逻辑删除）
     * DELETE /api/admin/product/delete/{id}
     * 需要管理员权限
     * 注意：逻辑删除通过调用productService.updateProductStatus(id, 0)实现，
     * 将status设为0，不是物理删除数据
     *
     * @param id      商品ID
     * @param request HTTP请求对象
     * @return 操作结果
     */
    @DeleteMapping("/delete/{id}")
    public Result<String> deleteProduct(@PathVariable Integer id,
                                         HttpServletRequest request) {
        // 第一步：验证管理员权限
        Result<?> authResult = checkAdmin(request);
        if (authResult != null) {
            return (Result) authResult;
        }

        // 第二步：逻辑删除——通过Service层调用updateProductStatus将status设为0
        String result = productService.updateProductStatus(id, 0);
        if (result.contains("下架")) {
            return Result.success("删除商品成功", "删除商品成功");
        } else {
            return Result.error(400, result);
        }
    }

    /**
     * 修改商品状态（上下架）
     * PUT /api/admin/product/status
     * 需要管理员权限
     *
     * @param params  包含id和status
     * @param request HTTP请求对象
     * @return 操作结果
     */
    @PutMapping("/status")
    public Result<String> updateProductStatus(@RequestBody Map<String, Integer> params,
                                               HttpServletRequest request) {
        // 第一步：验证管理员权限
        Result<?> authResult = checkAdmin(request);
        if (authResult != null) {
            return (Result) authResult;
        }

        // 第二步：参数校验
        Integer id = params.get("id");
        Integer status = params.get("status");
        if (id == null) {
            return Result.error(400, "商品ID不能为空");
        }
        if (status == null) {
            return Result.error(400, "状态不能为空");
        }

        // 第三步：调用Service修改状态
        String result = productService.updateProductStatus(id, status);
        if (result.contains("成功") || result.contains("上架") || result.contains("下架")) {
            return Result.success(result, result);
        } else {
            return Result.error(400, result);
        }
    }
}
```

**预期效果：** AdminProductController注入ProductService、UserService和ProductMapper。每个接口都先调用checkAdmin验证管理员权限。list()使用productMapper.findAllByPage()查询所有商品（不过滤status）。delete()调用productService.updateProductStatus(id, 0)实现逻辑删除（经过Service层）。非管理员访问返回403，参数校验失败返回400，操作成功返回200。所有代码使用`jakarta.servlet`。

### 9.5.2 补充OrderService方法

在 `OrderService.java` 中补充以下方法：

```java
package com.ecommerce.service;

import com.ecommerce.entity.Order;
import com.ecommerce.entity.PageResult;
import com.ecommerce.entity.Product;
import com.ecommerce.mapper.OrderMapper;
import com.ecommerce.mapper.ProductMapper;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

import java.math.BigDecimal;
import java.util.Date;
import java.util.List;

/**
 * 订单服务类
 */
@Service
public class OrderService {

    @Autowired
    private OrderMapper orderMapper;

    @Autowired
    private ProductMapper productMapper;

    /**
     * 创建订单
     */
    @Transactional
    public Order createOrder(Integer userId, Integer productId, Integer quantity,
                             String receiverName, String receiverPhone,
                             String receiverAddress) {
        // 第一步：验证商品是否存在
        Product product = productMapper.findById(productId);
        if (product == null) {
            throw new RuntimeException("商品不存在");
        }

        // 第二步：验证商品是否上架
        if (product.getStatus() == null || product.getStatus() != 1) {
            throw new RuntimeException("商品已下架");
        }

        // 第三步：验证库存是否充足
        if (product.getStock() < quantity) {
            throw new RuntimeException("库存不足，当前库存：" + product.getStock());
        }

        // 第四步：扣减库存（使用条件更新防止超卖）
        int stockRows = productMapper.decreaseStock(productId, quantity);
        if (stockRows == 0) {
            throw new RuntimeException("库存不足，下单失败");
        }

        // 第五步：增加销量
        productMapper.increaseSales(productId, quantity);

        // 第六步：生成订单号
        String orderNo = "ORD" + System.currentTimeMillis();

        // 第七步：冗余商品信息到订单
        Order order = new Order();
        order.setOrderNo(orderNo);
        order.setUserId(userId);
        order.setProductId(productId);
        order.setProductName(product.getName());
        order.setProductImage(product.getImage());
        order.setProductPrice(product.getPrice());
        order.setQuantity(quantity);
        order.setTotalPrice(product.getPrice().multiply(new BigDecimal(quantity)));
        order.setReceiverName(receiverName);
        order.setReceiverPhone(receiverPhone);
        order.setReceiverAddress(receiverAddress);
        order.setStatus(0);    // 待付款
        order.setCreateTime(new Date());
        order.setUpdateTime(new Date());

        // 第八步：插入订单记录
        int orderRows = orderMapper.insert(order);
        if (orderRows == 0) {
            throw new RuntimeException("创建订单失败");
        }

        return order;
    }

    /**
     * 查询我的订单
     */
    public List<Order> getMyOrders(Integer userId) {
        return orderMapper.findByUserId(userId);
    }

    /**
     * 查询所有订单（分页）
     *
     * @param page 当前页码
     * @param size 每页条数
     * @return 分页结果
     */
    public PageResult<Order> getAllOrders(int page, int size) {
        if (page < 1) {
            page = 1;
        }
        if (size < 1) {
            size = 10;
        }
        int offset = (page - 1) * size;
        List<Order> list = orderMapper.findByPage(offset, size);
        long total = orderMapper.count();
        return new PageResult<>(page, size, total, list);
    }

    /**
     * 修改订单状态
     *
     * @param id     订单ID
     * @param status 订单状态：0待付款 1已付款 2已发货 3已完成 4已取消
     */
    public void updateOrderStatus(Integer id, Integer status) {
        if (status == null || status < 0 || status > 4) {
            throw new RuntimeException("订单状态无效，0-待付款 1-已付款 2-已发货 3-已完成 4-已取消");
        }
        orderMapper.updateStatus(id, status);
    }

    /**
     * 按状态分页查询订单
     *
     * @param page   当前页码
     * @param size   每页条数
     * @param status 订单状态
     * @return 分页结果
     */
    public PageResult<Order> getAllOrdersByStatus(int page, int size, Integer status) {
        if (page < 1) {
            page = 1;
        }
        if (size < 1) {
            size = 10;
        }
        int offset = (page - 1) * size;
        List<Order> list = orderMapper.findByPageWithStatus(offset, size, status);
        long total = orderMapper.countByStatus(status);
        return new PageResult<>(page, size, total, list);
    }
}
```

**预期效果：** OrderService的createOrder返回Order对象，异常时抛出RuntimeException触发事务回滚。getAllOrders分页查询所有订单（参数为int类型）。updateOrderStatus修改订单状态（返回void，异常时抛出RuntimeException）。getAllOrdersByStatus按状态分页查询订单。订单状态码统一：0待付款/1已付款/2已发货/3已完成/4已取消。

## 9.6 AdminOrderController编写

"订单管理控制器和商品管理控制器的结构类似，每个接口都先验证管理员权限。"

在 `com.ecommerce.controller` 包下创建 `AdminOrderController.java`：

```java
package com.ecommerce.controller;

import com.ecommerce.entity.Order;
import com.ecommerce.entity.PageResult;
import com.ecommerce.entity.Result;
import com.ecommerce.entity.User;
import com.ecommerce.service.OrderService;
import com.ecommerce.service.UserService;
import jakarta.servlet.http.HttpServletRequest;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.web.bind.annotation.*;

import java.util.Map;

/**
 * 管理员-订单管理控制器
 */
@RestController
@RequestMapping("/api/admin/order")
public class AdminOrderController {

    @Autowired
    private OrderService orderService;

    @Autowired
    private UserService userService;

    /**
     * 验证管理员权限
     */
    private Result<?> checkAdmin(HttpServletRequest request) {
        Integer userId = (Integer) request.getAttribute("userId");
        if (userId == null) {
            return Result.error(401, "未登录");
        }
        User user = userService.getUserById(userId);
        if (user == null) {
            return Result.error(401, "用户不存在");
        }
        if (user.getRole() == null || user.getRole() != 1) {
            return Result.error(403, "无权限");
        }
        return null;
    }

    /**
     * 查询所有订单（分页，支持status筛选）
     * GET /api/admin/order/list?page=1&size=10
     * GET /api/admin/order/list?page=1&size=10&status=1
     * 需要管理员权限
     *
     * @param page    当前页码
     * @param size    每页条数
     * @param status  订单状态（可选，不传则查询所有）
     * @param request HTTP请求对象
     * @return 分页订单列表
     */
    @GetMapping("/list")
    public Result<PageResult<Order>> getAllOrders(
            @RequestParam(defaultValue = "1") int page,
            @RequestParam(defaultValue = "10") int size,
            @RequestParam(required = false) Integer status,
            HttpServletRequest request) {
        // 第一步：验证管理员权限
        Result<?> authResult = checkAdmin(request);
        if (authResult != null) {
            return (Result) authResult;
        }

        // 第二步：根据是否传入status选择查询方式
        PageResult<Order> pageResult;
        if (status != null) {
            pageResult = orderService.getAllOrdersByStatus(page, size, status);
        } else {
            pageResult = orderService.getAllOrders(page, size);
        }
        return Result.success("查询成功", pageResult);
    }

    /**
     * 修改订单状态
     * PUT /api/admin/order/status
     * 需要管理员权限
     *
     * @param params  包含id和status
     * @param request HTTP请求对象
     * @return 操作结果
     */
    @PutMapping("/status")
    public Result<String> updateOrderStatus(@RequestBody Map<String, Integer> params,
                                             HttpServletRequest request) {
        // 第一步：验证管理员权限
        Result<?> authResult = checkAdmin(request);
        if (authResult != null) {
            return (Result) authResult;
        }

        // 第二步：参数校验
        Integer id = params.get("id");
        Integer status = params.get("status");
        if (id == null) {
            return Result.error(400, "订单ID不能为空");
        }
        if (status == null) {
            return Result.error(400, "订单状态不能为空");
        }

        // 第三步：调用Service修改订单状态
        try {
            orderService.updateOrderStatus(id, status);
            return Result.success("订单状态修改成功", "订单状态修改成功");
        } catch (RuntimeException e) {
            return Result.error(400, e.getMessage());
        }
    }
}
```

**预期效果：** AdminOrderController注入OrderService和UserService。每个接口都先调用checkAdmin验证管理员权限。list()支持status参数筛选，传入status时调用getAllOrdersByStatus，不传时调用getAllOrders。updateOrderStatus()调用Service层方法，通过try-catch捕获异常。非管理员访问返回403。所有代码使用`jakarta.servlet`。

## 常见问题

| 序号 | 问题描述 | 可能原因 | 解决方案 |
|------|----------|----------|----------|
| 1 | 管理接口返回403 | 当前用户role不是1 | 确认数据库中用户role字段为1，或使用管理员账号登录 |
| 2 | 新增商品后列表查询不到 | 商品status未设置，默认为null | 检查addProduct方法是否设置了默认status=1 |
| 3 | 删除商品后再查询还能查到 | 逻辑删除只改status为0，前台列表WHERE status=1 | 这是正常行为，逻辑删除后商品不再出现在前台列表。管理后台list接口使用findAllByPage（不过滤status） |
| 4 | 修改商品时部分字段变成null | update方法使用了动态SQL，只更新非null字段 | 只传需要修改的字段，不传的字段不会被更新 |
| 5 | 修改订单状态返回400 | status值不在0-4范围内 | 确保status值为0、1、2、3、4之一 |
| 6 | 管理后台商品列表与前台不同 | 前台用findByPage（过滤status=1），后台用findAllByPage（不过滤status） | 管理后台需要看到所有商品包括下架的 |

## 本章小结

本章完成了管理后台的商品管理和订单管理功能：

1. **ProductMapper完整接口**：包含findAll、findByPage、findById、count、searchByKeyword、decreaseStock、increaseSales、insert、update、updateStatus、findAllByPage、countAll、findByCategoryPage、countByCategory、searchByKeywordFulltext共16个方法。逻辑删除统一使用updateStatus方法（将status设为0），不再使用deleteById。前台分页使用findByPage+count（过滤status=1），管理后台分页使用findAllByPage+countAll（不过滤status）
2. **ProductService管理方法**：addProduct、updateProduct、updateProductStatus均返回String类型的结果消息。addProduct设置默认值后插入，updateProduct先查后改，updateProductStatus校验状态值后修改（逻辑删除也使用此方法）
3. **AdminProductController**：注入ProductService、UserService和ProductMapper。每个接口都先调用checkAdmin私有方法验证管理员权限。list()使用productMapper.findAllByPage()查询所有商品（不过滤status）。delete()调用productService.updateProductStatus(id, 0)实现逻辑删除（经过Service层）
4. **OrderMapper完整接口**：包含insert、findByUserId、findAll、findByPage、count、updateStatus、findRecent、countToday、sumPaidTotalPrice、countByDate、selectCategorySales、findByPageWithStatus、countByStatus共13个方法
5. **OrderService管理方法**：getAllOrders(int page, int size)分页查询所有订单、updateOrderStatus(Integer id, Integer status)修改订单状态（返回void，异常时抛RuntimeException）、getAllOrdersByStatus(int page, int size, Integer status)按状态分页查询
6. **AdminOrderController**：注入OrderService和UserService。每个接口都先调用checkAdmin验证管理员权限。list()支持status参数筛选，传入status时调用getAllOrdersByStatus。updateOrderStatus()通过try-catch捕获Service层异常

**核心设计要点：**
- **逻辑删除**：删除商品不调用DELETE SQL，而是通过`productService.updateProductStatus(id, 0)`将status设为0，保留数据便于追溯。delete接口经过Service层而非直接操作Mapper
- **动态更新**：使用MyBatis的`<if>`标签实现只更新非null字段
- **权限验证**：每个管理接口先调用Controller内的checkAdmin私有方法验证role==1，返回Result.error(403, "无权限")
- **Service层返回类型**：ProductService管理方法返回String，OrderService的createOrder返回Order、updateOrderStatus返回void，异常时抛RuntimeException由Controller捕获
- **状态码统一**：订单状态0待付款/1已付款/2已发货/3已完成/4已取消，全手册一致
- **Jakarta EE**：SpringBoot 3.x使用jakarta.servlet替代javax.servlet
- **表名规范**：使用product/orders/user，无t_前缀
- **前后台查询区别**：前台用findByPage+count（过滤status=1），管理后台用findAllByPage+countAll（不过滤status）

**全书总结：** 第7-9章完整实现了花之恋鲜花电商平台的用户认证、商品展示、订单交易和管理后台四大核心模块。技术栈基于SpringBoot 3.x + MyBatis + JWT，采用前后端分离架构，统一响应格式，分层清晰（Controller → Service → Mapper），事务控制可靠，权限验证完整。
